# プロトタイプ差分メモ

最終更新: 2026-05-06

この文書は、現行の MonoGame プロトタイプと `docs/game-spec.md` の差分を整理するためのメモです。今後の実装は、ここをチェックリストとして更新します。`docs/game-spec.md` は 2026-05-05 に GDD 形式へ再整理済みです。

現行の公開対象は `RYZECHo.Prototype v0.0.3` で、完成版ではなく A/B 2 サイト構成の戦術スライスを安定配布するための継続更新版として扱います。`v0.0.3` の Windows 向け配布物、README、リリースノートは更新済みです。

## 現在できていること

### ルール / ラウンド

- 100 度視界を核にしたトップダウンの戦術スライス
- 戦闘フェーズでは全体俯瞰をやめ、プレイヤー周辺だけを映す近接カメラと濃い死角フォグを採用
- 初回構築フェーズ、ボス選択、投資、戦闘フェーズの大枠
- 固定 4v4 編成、7 ラウンド先取、6-6 以降の OT 継続
- 第 4 ラウンド終了時の攻守交代と再エディット
- A/B 2 サイトのボム設置、解除、爆破タイマーの基礎ループ

### 視界 / 音情報

- `SR` 装備時 80 度視野、足音 3 歩ごと、0.3 秒寿命、死角向け矢印、壁越し減衰の基礎反映
- 画面端インジケーターと味方共有を含む音情報、ミニマップの味方視界コーン、シェア・ビジョンの基礎反映
- 波紋 UI による簡易的な音可視化
- 10 秒静止時の呼吸波紋と呼吸音ペナルティ

### 経済 / ロードアウト

- メイン武器 9 種とサブウェポン 2 種の個別テーブル、価格、装弾数、予備弾、視認レンジ段階
- 4 人個別投資 UI、ボス 1 試合 2 回制限、300 円最適効率寄りの投資バフ、ボス撃破配当、敵ボス撃破 +800 円、`ULT +2`
- ULT ポイント進行、ボス別の投資額と ULT 状態表示、一覧型ショップ UI

### エージェント / スキル

- 構築フェーズ中 `6` で、ヴェール、ヴァイン、ニトロ、オアシス、ディバイド、グリッチを切替可能
- 戦闘中 `1 / 2 / 3` で、選択中エージェントのスキル 1 / スキル 2 / ULT を発動可能
- スキル購入制の導入と準備フェーズ UI との接続（実装済み）
- 試合中のアクション（キル、オーブ回収、ボム操作）から ULT ポイントおよびアカウント XP への変換ロジック
- 一時ワールド効果（毒霧、スモーク、ゾーン等）の視認性（Alpha値）、範囲、持続時間、クールダウンをプレイテスト値へ調整済み
- 瞬間加速、オーバードライブ、再生ナノマシン、オーバーシールド、ゴースト・ムーブなどのプレイヤー状態効果
- ソナー矢、ハンターズアイ、索敵ペット、警告センサーによる共有視界の拡張
- エージェントスキルによる一時設置物と一時エリア効果
- AI が敵味方のエージェント種別に応じて、索敵、煙幕、回復、移動、ロックダウン、システム妨害などを簡易判断で使用

### 構築 / マップエディット

- ノー・ビルド・ゾーン、トラップ近接制限、強化扉連結上限、主要ルート全封鎖防止、3 レーン維持チェック
- 各構築物（強化扉、カバー、トラップ等）の耐久力(Health)、APコスト、AIターゲット属性をプレイテスト値へ調整済み
- 破壊可能な構築物の管理と、AI による進路上の障害物検知ロジックの基礎
- 設置物スキン差し替えと、プレイを中断しない外周サイン型の会場広告テーマ差し替え基礎

### 進行 / マネタイズ / 整合性

- ローカル保存されるアカウント進行、レベル、ランク、エージェント・クレジット、契約進行の基礎
- 非 P2W のローカルコスメストア、設置物スキン、広告テーマ、バナー、キル演出 inventory
- 外周サイン型広告テーマ、広告露出カウント、コスメトークン報酬
- MatchProgressionService による試合中行動と報酬の紐付け実装
- 起動時の投資状態初期化クラッシュ修正と、署名不一致のローカル保存データを再署名で復旧するフェイルセーフ

## 主な未反映項目

### ルール

- A/B 2 サイトの固定タクティカルスライスで、可変マッププールや本番向けフロントエンド UI までは未反映

### エージェント

- スキル所持数（ストック制）や、ラウンド間での詳細な持ち越しルールの厳密な適用
- 各スキルの数値、範囲、持続時間、VFX はトップダウン検証用の仮実装で、最終仕様への調整が必要
- オーブ回収やエージェント固有ルートは未実装

### マップエディット

- レギュレーションは A/B 2 サイト向けの近似 3 レーン審査で、完全なマップ検証ツールまでは未実装

### 継続性とマネタイズ

- プロフィール保存はローカル JSON ベースの簡易実装で、アカウント基盤やオンライン同期は未実装
- 本番課金、広告配信 SDK、オンライン inventory 同期は未実装

### セキュリティと運用

- チート対策はクライアント内の整合性監視、保存データ署名検証、強制終了までで、サーバー権威の検証やオンライン制裁導線は未実装

## 実装の進め方

### フェーズ 1

- MonoGame プロトタイプ上のフェーズ 1 は完了扱い

### フェーズ 2

- AI のエージェント別スキル使用、スキル購入コスト、詳細演出を調整する
- オーブ回収や ULT 獲得イベントを試合中行動へ接続する

### フェーズ 3

- 完全なマップ検証、本番ストア / オンライン inventory、サーバー権威検証へ進む

## 次に詰める候補

- AI が進路上の構築物（強化扉など）を能動的に攻撃・破壊してサイトへ進入する挙動の強化
- ULTポイント獲得、オーブ回収時の演出（オーディオリップルとVFX）の強化
- 武器カテゴリごとの移動速度減衰、およびSR装備時の視野角制限(80度)の動的変動の厳密な適用

## 運用ルール

- 仕様が衝突した場合は `docs/game-spec.md` を優先
- 実装差分が埋まったら、この文書の該当項目を更新または削除する
- 仕様差分、リリース状態、既知制限に関わる `md` 更新はコード変更と同じコミットに含める
- `GameModel.cs` が 3000 行を超えたら、一度リファクタリングして複数ファイルへ分割する

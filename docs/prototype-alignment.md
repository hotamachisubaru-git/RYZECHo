# プロトタイプ差分メモ

最終更新: 2026-05-04

この文書は、現行の MonoGame プロトタイプと `docs/game-spec.md` の差分を整理するためのメモです。今後の実装は、ここをチェックリストとして更新します。

現行の公開対象は `RYZECHo.Prototype v0.0.3` で、完成版ではなく A/B 2 サイト構成の戦術スライスを安定配布するための継続更新版として扱います。`v0.0.3` の Windows 向け配布物、README、リリースノートは更新済みです。

## 現在できていること

- 120 度視界を核にしたトップダウンの戦術スライス
- 初回構築フェーズ、ボス選択、投資、戦闘フェーズの大枠
- 固定 4v4 編成、7 ラウンド先取、6-6 以降の OT 継続
- 第 4 ラウンド終了時の攻守交代と再エディット
- A/B 2 サイトのボム設置、解除、爆破タイマーの基礎ループ
- 波紋 UI による簡易的な音可視化
- `SR` 装備時 100 度視野、足音 3 歩ごと、0.3 秒寿命、死角向け矢印、壁越し減衰の基礎反映
- 画面端インジケーターと味方共有を含む音情報、シェア・ビジョンの基礎反映
- メイン武器 9 種とサブウェポン 2 種の個別テーブル、価格、装弾数、予備弾、視認レンジ段階
- 4 人個別投資 UI、ボス 1 試合 2 回制限、300 円最適効率寄りの投資バフ、ボス撃破配当、敵ボス撃破 +800 円、`ULT +2`
- ULT ポイント進行、ボス別の投資額と ULT 状態表示、一覧型ショップ UI
- ノー・ビルド・ゾーン、トラップ近接制限、強化扉連結上限、主要ルート全封鎖防止、3 レーン維持チェック、静止 10 秒後の呼吸音ペナルティ
- ローカル保存されるアカウント進行、レベル、ランク、エージェント・クレジット、契約進行の基礎
- 設置物スキン差し替えと、プレイを中断しない外周サイン型の会場広告テーマ差し替え基礎
- リコンビーコン、シールドリレーを含む破壊可能な構築物
- ランタイム整合性チェック、保存データ署名検証、重大な整合性違反検知時の強制終了
- 起動時の投資状態初期化クラッシュ修正と、署名不一致のローカル保存データを再署名で復旧するフェイルセーフ

## 主な未反映項目

### ルール

- A/B 2 サイトの固定タクティカルスライスで、可変マッププールや本番向けフロントエンド UI までは未反映

### エージェント

- 固定キャストのみで、ヴェール、ヴァイン、ニトロ、オアシス、ディバイド、グリッチのスキル未実装
- ULT ポイントはリソース進行までで、各エージェントの ULT 発動スキル本体は未実装
- オーブ回収やエージェント固有ルートは未実装

### マップエディット

- 設置物の種類は増えたが、完成版仕様にある全トラップ / 全防衛ギミックを網羅していない
- レギュレーションは A/B 2 サイト向けの近似 3 レーン審査で、完全なマップ検証ツールまでは未実装

### 継続性とマネタイズ

- プロフィール保存はローカル JSON ベースの簡易実装で、アカウント基盤やオンライン同期は未実装
- 空間内広告は外周サイン型の基礎のみで、試合進行を止める演出や画面占有型広告は入れていない
- ストア、課金導線、広告配信切替、本番用コスメ inventory は未実装

### セキュリティと運用

- チート対策はクライアント内の整合性監視、保存データ署名検証、強制終了までで、サーバー権威の検証やオンライン制裁導線は未実装

## 実装の進め方

### フェーズ 1

- 4v4 ラウンド制、ボムルール、攻守交代、OT 基礎まで反映済み
- オーディオ・リップルは `SR` 100 度視野、0.3 秒寿命、足音間引き、赤黄矢印、壁越し減衰の基礎まで反映済み
- MonoGame プロトタイプ上のフェーズ 1 は完了扱い

### フェーズ 2

- エージェント 6 名の実装
- 武器テーブルの個別化は MonoGame プロトタイプに反映済み
- ボス投資経済の本仕様化は MonoGame プロトタイプにコアループを反映済み
- サブウェポン、ULT ポイント、シェア・ビジョン、一覧型ショップ UI は MonoGame プロトタイプに反映済み
- エージェント 6 名の実装は Unity 側で進める前提

### フェーズ 3

- マップエディットの規制ルールは MonoGame プロトタイプに基礎反映済み
- A/B 2 サイト向けの 3 レーン維持チェックと設置物追加は MonoGame プロトタイプに反映済み
- 成長、ランク、アンロックは MonoGame プロトタイプにローカル進行として反映済み
- 収益化用の見た目差し替え基盤は、外周サイン型広告を含めて MonoGame プロトタイプに基礎反映済み
- ローカル整合性チェックと保存データ改ざん検知は MonoGame プロトタイプに反映済み

## 運用ルール

- 仕様が衝突した場合は `docs/game-spec.md` を優先
- 実装差分が埋まったら、この文書の該当項目を更新または削除する
- 仕様差分、リリース状態、既知制限に関わる `md` 更新はコード変更と同じコミットに含める
- `GameModel.cs` が 3000 行を超えたら、一度リファクタリングして複数ファイルへ分割する

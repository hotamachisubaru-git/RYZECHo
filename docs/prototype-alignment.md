# プロトタイプ差分メモ

最終更新: 2026-03-21

この文書は、現行の WinForms プロトタイプと `docs/game-spec.md` の差分を整理するためのメモです。今後の実装は、ここをチェックリストとして更新します。

## 現在できていること

- 120 度視界を核にしたトップダウンの戦術スライス
- 初回構築フェーズ、ボス選択、投資、戦闘フェーズの大枠
- 固定 4v4 編成、7 ラウンド先取、6-6 以降の OT 継続
- 第 4 ラウンド終了時の攻守交代と再エディット
- 1 サイトのボム設置、解除、爆破タイマーの基礎ループ
- 波紋 UI による簡易的な音可視化
- `SR` 装備時 100 度視野、足音 3 歩ごと、0.3 秒寿命、死角向け矢印、壁越し減衰の基礎反映
- メイン武器 9 種の個別テーブル、価格、装弾数、予備弾、視認レンジ段階
- ボス 1 試合 2 回制限、300 円最適効率寄りの投資バフ、ボス撃破配当、敵ボス撃破 +800 円
- ノー・ビルド・ゾーン、トラップ近接制限、強化扉連結上限、主要ルート全封鎖防止、静止 10 秒後の呼吸音ペナルティ
- ローカル保存されるアカウント進行、レベル、ランク、エージェント・クレジット、契約進行の基礎
- 設置物スキン差し替えと会場広告テーマ差し替えの基礎
- 破壊可能な構築物の一部

## 主な未反映項目

### ルール

- 1 サイト構成のタクティカルスライスで、マルチサイトや本番向けの攻守 UI までは未反映

### 視界と情報

- シェア・ビジョン未実装
- 音イベントは床面矢印ベースの簡易版で、画面端演出や味方共有との整合は未調整

### エージェント

- 固定キャストのみで、ヴェール、ヴァイン、ニトロ、オアシス、ディバイド、グリッチのスキル未実装
- ULT ポイント進行、オーブ回収、敵ボス撃破 +2 未実装

### 武器

- サブウェポン 2 種の個別選択、購入、持ち替えは未実装
- 武器選択 UI は Q/E 切替ベースの簡略版で、ショップ一覧 UI までは未反映

### ボス経済

- 4 人個別投資 UI は未実装で、WinForms 版では `_selectedBet` をチーム総投資額として近似
- ULT ポイント系そのものは未実装で、敵ボス撃破時の `ULT +2` は未反映

### マップエディット

- 設置物の種類が限定的
- レギュレーションは現行 1 サイトマップ向けの簡易ヒューリスティックで、完全な 3 レーン審査までは未実装

### 継続性とマネタイズ

- プロフィール保存はローカル JSON ベースの簡易実装で、アカウント基盤やオンライン同期は未実装
- ストア、課金導線、広告配信切替、本番用コスメ inventory は未実装

## 実装の進め方

### フェーズ 1

- 4v4 ラウンド制、ボムルール、攻守交代、OT 基礎まで反映済み
- オーディオ・リップルは `SR` 100 度視野、0.3 秒寿命、足音間引き、赤黄矢印、壁越し減衰の基礎まで反映済み
- WinForms プロトタイプ上のフェーズ 1 は完了扱い

### フェーズ 2

- エージェント 6 名の実装
- 武器テーブルの個別化は WinForms プロトタイプに反映済み
- ボス投資経済の本仕様化は WinForms プロトタイプにコアループを反映済み
- エージェント 6 名の実装は Unity 側で進める前提

### フェーズ 3

- マップエディットの規制ルールは WinForms プロトタイプに基礎反映済み
- 成長、ランク、アンロックは WinForms プロトタイプにローカル進行として反映済み
- 収益化用の見た目差し替え基盤は WinForms プロトタイプに基礎反映済み

## 運用ルール

- 仕様が衝突した場合は `docs/game-spec.md` を優先
- 実装差分が埋まったら、この文書の該当項目を更新または削除する
- `GameModel.cs` が 3000 行を超えたら、一度リファクタリングして複数ファイルへ分割する
